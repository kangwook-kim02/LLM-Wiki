# /lint — LLM Wiki 건강 검사

`wiki/` 전체를 스캔하여 모순·고아 페이지·유령 링크 등을 탐지하고 수정 여부를 확인한다.

---

## 절차

### Step 1 — 전체 페이지 수집
`wiki/` 하위 모든 `.md` 파일 목록을 수집한다 (`index.md`, `log.md` 제외).

### Step 2 — 각 페이지 읽기
모든 페이지를 읽고 다음 항목을 탐지한다.

| 항목 | 탐지 기준 |
|------|----------|
| **모순** | 동일 개념에 대해 서로 상충하는 정의·주장 |
| **고아 페이지** | 다른 어떤 페이지에서도 `[[링크]]`되지 않은 페이지 |
| **유령 링크** | `[[링크]]`는 있지만 대응하는 파일이 없는 경우 |
| **누락된 교차 참조** | A가 B를 언급하는데 B가 A를 역참조하지 않는 경우 |
| **구식 클레임** | 이후 인제스트된 소스가 기존 내용을 무효화하는 경우 |
| **빈 섹션** | 템플릿 헤더만 있고 내용이 없는 섹션 |

### Step 3 — 결과 보고
발견 사항을 심각도별로 분류하여 보고한다:

```
🔴 심각 (즉시 수정 필요)
  - 모순, 잘못된 출처

🟡 경고 (수정 권장)
  - 고아 페이지, 유령 링크, 누락된 교차 참조

🟢 정보 (참고용)
  - 아직 페이지가 없는 언급된 개념 목록
  - 빈 섹션
```

사용자에게 각 항목을 수정할지 물어본다.

### Step 4 — 승인된 항목 수정
사용자가 수정을 승인한 항목만 수정한다.

### Step 5 — 로그 기록
`wiki/log.md` 맨 아래에 append한다:

```markdown
## [오늘날짜] lint | 발견 N건 (심각 X, 경고 Y, 정보 Z)

- 수정된 항목: (목록)
- 미수정 항목: (목록)
```

---

## 출력

- 발견 사항 전체 목록
- 수정 여부 확인 후 처리 결과
