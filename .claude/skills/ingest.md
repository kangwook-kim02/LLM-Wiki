# Skill: ingest

## 입력

| 변수 | 설명 | 예시 |
|------|------|------|
| `$SOURCE` | 사용자 메시지에서 추출한 파일명 | `10. New Topic.pdf` |

---

## 사전 확인

1. `raw/$SOURCE` 파일이 존재하는지 확인한다.
   - 없으면: `"raw/$SOURCE 파일을 찾을 수 없습니다. raw/ 폴더에 파일이 있는지 확인해 주세요."` 로 중단.
2. `wiki/index.md`를 읽어 현재 소스 번호(NN)를 파악한다.
   - 다음 소스 번호 = 현재 소스 수 + 1 (두 자리 zero-pad: 01, 02, ...)

---

## 절차

### Step 1 — 소스 읽기
`raw/$SOURCE` 를 Read 도구로 읽는다.

### Step 2 — 분석
다음을 추출한다:
- **핵심 주장** 3~5개 (bullet)
- **새로 등장한 개념** 목록
- **등장한 엔티티** (도구, 인물, 프레임워크) 목록
- **주목할 인사이트** 1~3개

### Step 3 — 소스 요약 페이지 생성
`wiki/sources/source-NN-[slug].md` 를 생성한다.
- `[slug]` = 파일명을 소문자 kebab-case로 변환
- CLAUDE.md 소스 요약 템플릿 사용

### Step 4 — 개념 페이지 생성·업데이트
Step 2에서 추출한 개념마다:
- 기존 페이지(`wiki/concepts/`)가 있으면 → 새 내용 병합, 출처 추가
- 없으면 → CLAUDE.md 개념 템플릿으로 신규 생성

### Step 5 — 엔티티 페이지 생성·업데이트
Step 2에서 추출한 엔티티마다:
- 기존 페이지(`wiki/entities/`)가 있으면 → 업데이트
- 없으면 → CLAUDE.md 엔티티 템플릿으로 신규 생성

### Step 6 — 주제 페이지 업데이트
`wiki/topics/` 중 이 소스와 관련된 페이지를 찾아 업데이트한다.
관련 주제가 없고 새 주제가 필요하면 신규 생성한다.

### Step 7 — 인덱스 업데이트
`wiki/index.md`의 소스 테이블, 개념/엔티티/주제 테이블, 통계를 업데이트한다.

### Step 8 — 로그 기록
`wiki/log.md` 맨 아래에 append한다:

```markdown
## [오늘날짜] ingest | $SOURCE

### 처리된 소스
- `wiki/sources/source-NN-[slug].md`

### 생성·업데이트된 페이지
- (생성된 페이지 목록)

### 모순 플래그
- (기존 위키와 충돌하는 내용이 있으면 여기에 명시, 없으면 "없음")
```

---

## 출력

- 생성·수정된 파일 목록을 사용자에게 보고한다.
- 모순이 발견된 경우 명시적으로 알린다.
- 아직 페이지가 없는 관련 개념이 있으면 "다음 인제스트 시 추가 권장" 목록을 제시한다.

---

## 원칙

- 한 소스 인제스트 시 **10~15개 페이지**를 건드리는 것이 정상이다.
- `raw/` 파일은 절대 수정하지 않는다.
- 출처(`[[source: 파일명]]`) 없는 주장을 작성하지 않는다.
- 기존 내용과 모순될 경우 덮어쓰지 말고 **양쪽을 병기하고 플래그**한다.
