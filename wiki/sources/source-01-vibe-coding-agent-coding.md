# Vibe Coding and Agent Coding

**파일:** `raw/1. Vibe coding and Agent coding.pdf`
**인제스트 날짜:** 2026-05-24
**유형:** 강의자료 (CSE3308 Practical Session 01)

## 핵심 주장

- **소프트웨어 엔지니어링의 진화**: 하드웨어 종속(1940~) → 추상화/프레임워크(~2024) → 지능형 시스템 시대(2024~)로 패러다임이 전환되었다.
- **Vibe Coding(2024.4Q~2025.3Q)**: 자연어로 목표와 제약을 전달하고, LLM이 코드를 생성하는 방식. 개발자의 역할이 코드 작성에서 청사진 설계·프롬프트 엔지니어링·검증으로 전환된다.
- **Vibe Coding의 한계**: 코드 규모가 커질수록 일관성/검증/운영에서 기술 부채가 빠르게 쌓인다.
- **Agentic Coding(2025.4Q~)**: 코딩 에이전트를 통해 SDLC가 보장되는 개발 프로세스를 수행 가능. Vibe Coding의 단점을 보완.
- **System Analysis의 역할**: 사용자/운영/비즈니스가 원하는 목표를 도출하고, 시스템 범위·제약을 명확화하며, 요구사항을 모델/명세로 변환하여 검증 가능하게 만드는 것.
