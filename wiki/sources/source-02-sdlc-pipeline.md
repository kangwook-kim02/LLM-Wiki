# SDLC Pipeline in Vibe Coding

**파일:** `raw/2. SDLC pipeline in Vibe coding.pdf`
**인제스트 날짜:** 2026-05-24
**유형:** 강의자료 (CSE3308 Week 2)

## 핵심 주장

- **체계적 개발의 필요성**: Vibe Coding으로 단순 도구는 빠르게 만들 수 있지만, 복잡한 시스템에서는 체계적인 Requirements 없이 제대로된 결과물을 낼 수 없다.
- **Spec-Driven Development**: Vibe Coders의 패턴은 "Spec First, Code Later". PRD → Plan → SRS → User Story 순서로 작업을 분해한다.
- **Single/Sequential/Parallel Agent 구분**: 작업 복잡도에 따라 단일 → 순차 → 병렬 에이전트 구조로 확장한다.
- **SDLC 단계의 재해석**: Planning → Analysis → Design → Implementation 단계를 AI 협업 방식으로 재구성한다.
- **Agentic Coding의 인간 역할**: 프롬프트 엔지니어에서 Orchestrator로 변화.

## 주목할 인사이트

- Quick Sort 시각화 도구 예시: PRD 요소를 프롬프트에 포함시키자 결과물이 월등히 향상됨. "단순한 도구 하나도 체계적인 Requirements가 없으면 기대하는 결과를 내기 힘들다."
- 지하철 스크린도어 시스템 예시: 탑승객 관점과 시스템 관점의 요구사항이 완전히 다름 → 이해관계자 분석의 중요성.
- Sequential Agent: "각 에이전트는 독립적으로 작동하되 앞 단계가 완료되어야 다음 단계가 시작." SDLC Analysis 단계에서 수행하는 작업 분해와 동일하다.
